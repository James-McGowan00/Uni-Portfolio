Mario

My second assignment was creating a recreation of an old Mario level from 1983 called Mario’s Cement Factory.

Characters

To create Mario and Luigi I first loaded them in from a sprit sheet. I did this by creating a mesh/ texture that would load Mario and Luigi when the game launched. This would allow Mario and Luigi to appear on the screen when the game launched. One change I made to Mario and Luigi was the height of the characters from the original design I made them taller than the original game. I made them move with different key bindings, so 2 people can play at the same time as well as making them face in the direction the player pressed.

Movement

I added movement to Mario and Luigi which allows them to both jump and move with individual controls. The jump allows both Mario and Luigi to jump to the next level to avoid the Koppers. With the side to side movement this allows both Mario and Luigi to wrap around the map transporting to the other side. Mario and Luigi both have the ability to hit the POW block in the middle, this would stun the Koppers for a short person of time allowing them to kill them and collect points. Both Mario and Luigi can get hit by the Koppers and eventually die.

Sound implementation

To create the sound I used the built in SDL mixer which allowed me to access gMusic, this would then allow me to load in the music file at the launch of the game. The Mario theme tune will play once the player launches the game. This will continuously loop through until the player ends the game.

Menu

To create my own title screens I used a screen manager which would load the relevant screen when the player presses a button. To load the different screens I used a texture loader this would look for the image in the file, loading the map and other screen on to the SDL window. Using the screen managed I created a screen with controls on so the player could understand how to move Mario and Luigi. This was also used for the start screen I then further implemented a game over screen.

  • 12 weeks to develop
  • Visual Studio with SDL plugin
  • C++

DNA project

This was a project that I did in a pair, me and another student decided to create a puzzle game within UE4. We started off by putting our ideas forwards and doing research into how to create unblock me puzzles and Simon says. The first level was a tutorial maze for the player to get used to the controls, we both worked on that splitting up the level design, mechanics and UI.

The second level was the level I worked on, this was an unblock me puzzles. I stated by researching into different unblock me puzzles trying to solve them along the way.

Movement

I used the movement form the FPS template in UE4 as all I need was forwards, back, left and right.

Moving of blocks

I started off by creating a pick up system that would allow the player to pick up the blocks and move them around. I did this with the first person template and the gun that is available. I create a blueprint that would ray trace and when it hit an object it would attract that the the players gun meaning if they moved around it would follow, to fix this I had to lock the rotation and nonevent in X or Y depending on which way the clock was moving.

Building of puzzles

To create the puzzles I created blocks that was the height of the player and used previously unblock me puzzles to create my own I had to create them then test them to see if they were solve able. Once I got one puzzle set up I decide to create 3 more each getting harder as the player proceeds. Once I created all the puzzles I had problem trying to complete them as the blocks where the height of the player. I decide to shrink the block to a 1/3 of the size so the player would be able to jump into the centre of the puzzle and move blocks that were harder to reach.

Camera angles

The final thing I added was camera angles above the puzzle so the player could switch to them to see how they are getting on or use it as help to see what they could move next.

  • 2 weeks to develop
  • Unreal Engine
  • Blueprints
  • Working in a pair

Tower Defence

This was an assignment where I was able to create a tower defence game in a program of my choice. I decided to do it in UE4 as I have had the most experience in it.

AI

I started off by creating a AI spawner this would spawn waves of enemies which would run to an end point. To get them to run I first had to create a runner, this would include a behaviour tree and blackboard. This would give the AI instruction in what to do, in this case they would run to my end point set up within the spawner. To create different AI I created a database this would allow me to change the health, speed and the amount of money the player would get if the AI was killed.

Towers

To create the towers I started by creating a base actor, using a shape as the mesh. As well as this I added a shooting system that uses a sphere collision to detect when a AI enters, once the AI enters the tower it will begin shooting that target until the target is killed or leaves the collision sphere. I did this multiple times with different towers to do this I created a database which would allow me to alter the radius, fire rate, cost to place and how much damage the tower will deal to the AI. I also included a homing and non-homing projectile. The homing projectile will hit the AI and never miss, where as the non-homing would hit and cause a splash damage effect.

Mechanics

I had to create a method in order for the player to be able to place towers. To do this I would do a check to see if there was anything under the mouse, if not the player would be able to place a tower indicated by a green outline of the tower. Whereas if the check comes back with a tower underneath the player won’t be able to place a tower this is then indicated with a red outline of the tower.

Level design

I created 3 levels for my game. I had inspiration from York, Tower bridge and Stonehenge. With the York map I wanted it be close quarter with building hanging over the road and pavement. I also implemented a wall like the one that’s runs near the train station. I added my own features like a water fountain and castle gate. Next I created Tower bridge, this was meant to be easier than the first level as the player could see the AI running at them as they were coming from the opposite side of the bridge. Finally I created Stonehenge this was my hardest level as the AI was coming from every direction. This would mean the player would have to watch behind them at all times.

Money and lives

To create the money and lives I had to create a UI that would display this to the screen. I used the AI spawner system to detect once they had reached the end point by doing this I could then edit the lives of the player. Once the AI was deleted it would lower the players lives by the amount set in the database. This was kind of similar to the money situation, when I was checking if anything was under the mouse, I would take off the money set in the database when the tower was placed. I also did checks to see if the player had enough money to but the towers if not the button wouldn’t work. I added a feature where once the tower is placed they can’t get the money back. For the player to get money I had it set up for when the tower killed an AI they would gain money back also set in the database.

Menu system

Finally I created a start, pause, round, quit, win and lose screen. The start screen would allow the player to see the controls on how to play the game, this was also set up within the pause menu in case they forgot and or want a break. The round screens happen before each round, it gives the player time to place down there towers before beginning the round. The quiz screen is used to quit the game. The lose screen happens when the player loses all the lives. The win screen is a bit different as the player has to survive 3 levels with 10 round within and once they complete a level they must find a door to get to the next level and to win.
